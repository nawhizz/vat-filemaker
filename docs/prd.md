## 1. 개요

### 1.1. 목적

본 문서는 개인 사업자를 위한 부가세 신고 도우미 프로그램의 요구사항을 정의합니다. 이 프로그램은 엑셀 파일로 된 카드 사용 내역을 불러와 SQLite 데이터베이스에 저장하고, 국세청 '사업자등록상태조회'를 통해 사업자 상태를 확인하여 부가세 신고 대상/제외 대상을 관리하며, 최종적으로 부가세 신고를 위한 전자 매체 파일을 생성하는 것을 목표로 합니다.

### 1.2. 범위

본 프로그램은 Windows, macOS 운영체제를 지원하는 데스크톱 애플리케이션으로 개발됩니다.

### 1.3. 대상 사용자

본 프로그램의 주요 대상 사용자는 개인 사업자입니다.

## 2. 기능 요구사항

### 2.1. 사용자 정보 등록

- 사용자의 개인사업자 정보를 입력받아서 SQLite 데이터베이스에 저장하는 기능
- 저장된 개인사업자 정보를 조회하고 수정하는 기능

### 2.2. 엑셀 파일 업로드

-   카드사에서 다운받은 엑셀 파일을 업로드하는기능
-   다양한 엑셀 파일 형식 지원 (.xls, .xlsx)
-   업로드할 엑셀 데이터를 읽어서 미리보기 기능 제공

### 2.3. 데이터베이스 저장

-   읽어들인 엑셀 데이터를 SQLite 데이터베이스에 저장하는 기능
-   데이터 저장 시 오류 검사 및 예외 처리

### 2.4. 카드 내역 조회 및 편집

-   저장된 카드 내역을 테이블 형태로 조회하는 기능
-   카드 내역 필터링 및 검색 기능 제공 (날짜, 카드사, 거래처 등)
-   카드 내역 수정 및 삭제 기능 제공

### 2.5. 사업자 상태 조회 및 관리

-   국세청 '사업자등록상태조회' API를 이용하여 사업자 상태(계속/폐업, 과세유형)를 확인하는 기능
-   사업자등록번호를 기준으로 사업자 상태 정보 관리
-   사업자 상태 변경 이력 관리

### 2.6. 부가세 제외 대상 거래 관리

-   사업자 상태(폐업, 간이과세, 면세)에 따른 자동 제외 처리
-   병원, 약국, 학원 등 별도 조건에 따른 제외 처리 기능 제공
-   제외 사유 입력 및 관리 기능 제공

### 2.7. 전자 매체 파일 생성

-   국세청 부가세 신고 양식에 맞는 전자 매체 파일 생성 기능
-   생성된 파일 미리보기 기능 제공
-   파일 저장 경로 설정 기능 제공

### 2.8. 통계 및 보고서

-   월별/분기별 카드 사용 내역 통계 기능 제공
-   부가세 신고 관련 보고서 생성 기능 제공 (선택 사항)
-   데이터 시각화 기능 제공 (선택 사항)

## 3. 비기능 요구사항

### 3.1. 사용성

-   직관적인 사용자 인터페이스
-   쉬운 데이터 입력 및 수정
-   빠른 데이터 처리 속도

### 3.2. 신뢰성

-   데이터 손실 방지
-   정확한 계산 결과
-   안정적인 프로그램 동작

### 3.3. 보안성

-   개인 정보 보호
-   데이터 암호화 (선택 사항)
-   안전한 API 연동

### 3.4. 성능

-   빠른 실행 속도
-   최소한의 시스템 자원 사용
-   대용량 데이터 처리 가능

## 4. UI/UX 요구사항

### 4.1. 디자인 컨셉

-   심플하고 직관적인 디자인
-   사용자 친화적인 인터페이스
-   세련된 색상 및 레이아웃

### 4.2. 주요 화면 구성

-   메인 화면: 카드 내역 요약 정보, 주요 기능 바로가기
-   카드 내역 조회/편집 화면: 카드 내역 테이블, 필터링 및 검색 기능
-   사업자 상태 관리 화면: 사업자 정보 조회 및 관리 기능
-   부가세 제외 대상 관리 화면: 제외 대상 설정 및 관리 기능
-   전자 매체 파일 생성 화면: 파일 생성 옵션 설정 및 실행 기능
-   통계/보고서 화면: 통계 데이터 시각화 및 보고서 생성 기능 (선택 사항)

## 5. 기술 요구사항

### 5.1. 개발 언어

-   Python

### 5.2. GUI 프레임워크

-   PySide6 + ModelView Architecture + PySide6-Fluent-Widgets 패키지

### 5.3. 데이터베이스

-   SQLite + SQLAlchemy (ORM)

### 5.4. API

*   국세청 '사업자등록상태조회' API

## 6. 향후 확장 기능

*   Next.js + FastAPI를 개발환경으로하는 웹기반 프로그램으로 전환
*   데이터베이스 처리 부분을 향우 FastAPI로 전환이 용이하도록 설계/개발 필요
